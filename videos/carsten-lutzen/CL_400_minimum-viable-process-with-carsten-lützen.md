# Minimum Viable Process with Carsten Lützen

## Summary
Carsten shares his recommended starting point for teams new to agile: a lightweight two-step loop of setting a clear Sprint Goal and running a Retrospective. This Minimum Viable Process strips agile down to its essential feedback loop, making it accessible for any team regardless of domain.

## Methods & Techniques

### Minimum Viable Process Loop (ab 00:20)
- **What:** A two-step iterative process — set a measurable goal for two weeks, then run a retrospective to evaluate and set the next goal. Repeat.
- **When to use:** When starting a new team or introducing agile practices for the first time; useful in any domain, not just software.
- **Group size:** Any team size
- **Duration:** Two-week cycles
- **How it works:**
  1. Define a clear goal for the next two weeks (inspired by the Scrum Sprint Goal)
  2. Apply SMART principles or an equivalent to make the goal measurable and concrete — specify which features or outcomes to validate, not just "all features done"
  3. At the end of the two weeks, validate: Did you achieve the goal? If not, what was achieved?
  4. Run a Retrospective: what worked, what didn't?
  5. Use retrospective insights to set the next two-week goal
  6. Repeat, and evolve toward Kanban, full Scrum, or other approaches as the team matures
- **Source:** https://www.youtube.com/watch?v=0sJyaBWtJdw&t=20

## Key Principles
- Start with the minimum: a clear goal and a retrospective are enough to create a functioning feedback loop
- Goals must be tangible and validatable — "all features implemented" is too vague; name the specific features and what done looks like
- The process should evolve from this foundation, not be imposed fully-formed from day one
- Domain-agnostic: the loop works in software, but also in other fields

## Facilitation Tips
- Don't over-engineer the first iteration — resist the urge to introduce a full Scrum setup immediately
- Make the Sprint Goal concrete enough that the team can unambiguously answer "did we achieve it?" after two weeks
- Use the retrospective output directly as input for the next goal — close the loop explicitly

## Best Used When
- Onboarding a new team to agile practices
- Introducing a lightweight process without overwhelming the team with ceremony
- Teams outside software development who want an iterative approach without full Scrum overhead
