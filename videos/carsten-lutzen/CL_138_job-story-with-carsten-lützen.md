# "Job Story" with Carsten Lützen

## Summary
Carsten introduces Job Stories as an alternative format to User Stories for writing backlog items. The format centers on situation, motivation, and expected outcome rather than a specific user persona — making it especially useful when multiple user types share the same goal or when writing epics.

## Methods & Techniques

### Job Stories (ab 00:10)
- **What:** A backlog item format structured as: "When [situation], I want [motivation], so that [expected outcome]." Unlike User Stories, it omits a named user persona and focuses on context and outcome instead.
- **When to use:** When many different user types or personas perform the same action and want the same outcome; when writing epics that describe larger situations rather than granular tasks.
- **Group size:** N/A — a writing/refinement technique
- **Duration:** N/A — can be applied during backlog refinement sessions
- **How it works:**
  1. Define the triggering **situation** ("When I log in…")
  2. Describe the **motivation** ("…I want to see what's new…")
  3. State the **expected outcome** ("…so that I am up to date.")

## Key Principles
- Job Stories deliberately omit the user persona — this is both a strength and a limitation.
- The strength: they are outcome- and behavior-centric, which works well when multiple personas share the same need.
- They are particularly effective for **epics**, where the situation is broad and the item will later be broken down into smaller pieces (e.g., user stories).
- Carsten recommends focusing on the behavior and system you want to capture, then choosing the format (job story vs. user story) that best fits.

## Facilitation Tips
- Use Job Stories during backlog refinement or epic definition when the team struggles to pin down a single persona for a widely-shared action.
- When a Job Story is too large to implement directly (e.g., "When a user submits an order…"), use it as an epic anchor and break it down into user stories or other smaller items.
- Job Stories can support prioritization discussions by keeping focus on outcomes and motivations rather than who is doing the action.

## Best Used When
- Multiple user types or personas perform the same action with the same expected outcome.
- Writing or refining epics that describe high-level system behavior.
- The team wants to move away from persona-centric thinking and focus on jobs-to-be-done.

## Related Concepts
- **User Stories** — the more common alternative format; Carsten covered this in the previous video in the series.
- **Jobs-to-be-Done (JTBD)** — the broader framework that Job Stories are derived from.
- **Epics** — Job Stories are well-suited as a format for writing epics before breaking them down.
